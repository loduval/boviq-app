☁️ Architecture stockage centralisé

GitHub API · Commit d6db446

Les données voyagent vers le cloud !

Auparavant, toutes les données BOVIQ étaient stockées uniquement dans le localStorage de votre navigateur — ce qui signifiait que si vous changiez de navigateur ou d'appareil, tout était perdu. Désormais, chaque modification est automatiquement sauvegardée sur GitHub.

🔑
Configurer votre PAT
Dans BOVIQ V6, cliquez sur le badge de synchronisation (barre latérale) → entrez votre Personal Access Token GitHub. Le PAT est stocké uniquement dans votre navigateur.
💾
Sauvegarde automatique
Chaque modification déclenche une sauvegarde après 1,5 seconde d'inactivité. Le badge indique l'état : ✓ Synchronisé, ⏳ Sauvegarde en cours, ou ⚠️ Erreur.
🔄
Accès multi-appareils
Ouvrez BOVIQ sur n'importe quel appareil : les données sont chargées depuis data/boviq-data.json sur GitHub au démarrage. Fallback local si hors ligne.

🥛 Bridge MilKlic ↔ BOVIQ V6

localStorage partagé
🔗
Les données de contrôle laitier apparaissent maintenant sur le tableau de bord V6

Auparavant, les données importées dans l'application MilKlic restaient isolées. Désormais, les fonctions saveML() et loadML() écrivent simultanément dans localStorage['BOVIQ_MILKLIC'], permettant à BOVIQ V6 d'afficher immédiatement les KPIs laitiers sur le tableau de bord.

📍 Pour activer : Ouvrez d'abord boviq-milklic.html → importez votre CSV MilKlic → revenez sur BOVIQ V6. Les données s'affichent automatiquement sur le tableau de bord.

🔧 Corrections apportées

3 bugs · cette session
📊
Graphiques du tableau de bord — courbes vides
La fonction loadChartJs() retourne une Promise asynchrone mais était appelée de manière synchrone. Corrigé avec async/await pour garantir que Chart.js est chargé avant le rendu des courbes.
📈
Widget cours — courbe des prix illisible
La sparkline précédente ne prenait pas en compte l'échelle des données. Remplacée par une vraie courbe Chart.js avec axe Y borné min±2€ autour de la valeur centrale, rendant les variations lisibles.
🥛
Contrôle laitier — « données non disponibles » sur V6
Le bridge entre MilKlic et BOVIQ V6 était manquant. Ajout des écritures dans localStorage['BOVIQ_MILKLIC'] dans les fonctions saveML() et loadML() du module MilKlic.
